NAT 类型说明

网络地址转换（NAT）是现代网络连接互联网的基础。它会在不同网络之间转发数据包时转换源或目的 IP 和端口，让本地网络中的多个设备共享一个公网 IP 地址，节省 IPv4 资源，并在内部设备与外部网络之间增加一层隔离。

不同 NAT 类型主要影响外部设备能否主动连回你的设备，以及 P2P 打洞、联机游戏、视频通话、远程访问等场景的成功率。一般来说，限制越少，直连成功率越高；限制越强，越可能需要服务器中继。

NAT0 / Direct（无 NAT / 直连）

设备可直接使用公网地址通信，或已建立直连路径，外部连接限制最少，连通性通常最好。

Full Cone NAT（完全锥型 NAT）

内网地址和端口映射到固定公网端口后，任意外部主机都可通过该映射回连，P2P 兼容性最好。

Restricted Cone NAT（地址受限锥型 NAT）

公网端口映射相对固定，但只允许内网设备访问过的外部 IP 回连，兼容性较好。

Port Restricted NAT（端口受限锥型 NAT）

只允许内网设备访问过的外部 IP 和端口组合回连，P2P 成功率低于地址受限锥型 NAT。

Symmetric NAT（对称型 NAT）

每个目标独立映射端口，P2P 打洞困难，常需中继。

检测结果准确吗？

本工具基于浏览器 WebRTC ICE 候选和 STUN 探测结果进行分析，通常能较准确地反映当前网络路径的 NAT 类型。但受浏览器安全策略、候选信息暴露范围和网络环境影响，某些情况下可能无法完全区分 Restricted Cone 与 Port Restricted Cone。如需更精确的检测，建议配合专业 STUN 客户端或网络诊断工具交叉验证。

这个检测会收集我的数据吗？

不会。检测过程主要在您的浏览器本地完成，STUN 服务器仅用于返回当前网络的公网 IP 和端口映射信息。本站不会记录、存储或额外传输您的个人网络信息。
